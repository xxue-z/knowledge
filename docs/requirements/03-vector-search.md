# 需求文档：向量数据库与语义检索

## 1. 需求概述

向量数据库用于存储个人会话问答历史，支持语义检索功能。

---

| VEC-002 | 每条记录绑定 user_id（用户隔离） | README.md 第4节 |

---

## 3. 数据模型需求

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | UUID | 主键 | 会话唯一标识 |
| user_id | String | 索引 | 用户ID |
| dept_id | String | 索引 | 部门ID |
| question | Text | - | 用户问题 |
| answer | Text | - | 回答内容 |
| embedding | Vector | - | 问题向量嵌入 |
| sensitivity | String | enum | 敏感度级别 |
| source_refs | JSON | - | 来源引用列表 |
| created_at | DateTime | - | 创建时间 |

---

## 4. 非功能需求

| 需求编号 | 需求描述 |
|----------|----------|
| VEC-NFR-001 | 语义检索响应时间 < 1000ms |
| VEC-NFR-002 | 支持 100000+ 会话记录存储 |
| VEC-NFR-003 | 向量相似度计算精度 > 90% |
