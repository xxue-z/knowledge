# 异常码体系文档

## 1. 概述

本文档定义了知识平台系统的异常码体系，用于统一前后端的错误处理。异常码采用五位数格式，前两位表示模块编号，后三位表示异常编号。

## 2. 异常码格式

```
XXYYY
│ └── YYY: 异常编号（001-999）
└── XX: 模块编号（00-99）
```

## 3. 模块编号表

| 模块编号 | 模块名称 | 说明 |
|----------|----------|------|
| **00** | 系统运行异常 | 运行时错误、数据库连接失败、服务不可用等 |
| 10 | 系统配置模块 | 系统状态、配置管理、初始化等 |
| 11 | 认证模块 | 登录、token、权限等 |
| 12 | Wiki 模块 | Wiki 文档 CRUD |
| 13 | 问答模块 | 智能问答相关 |
| 14 | 知识导航模块 | 导航树管理 |
| 15 | 管理模块 | 管理员操作、审计日志 |
| 16 | 存储模块 | 对象存储操作 |
| 17 | 标签模块 | 标签管理 |
| 18 | 切片规则模块 | 文档切片规则 |
| 99 | 通用模块 | 通用错误（如参数校验失败） |

## 4. 异常码详细定义

### 4.1 00xxx - 系统运行异常

| 异常码 | 描述 | HTTP 状态码 | 说明 |
|--------|------|-------------|------|
| 00001 | 数据库连接失败 | 503 | PostgreSQL 连接失败 |
| 00002 | Redis 连接失败 | 503 | Redis 缓存连接失败 |
| 00003 | Milvus 连接失败 | 503 | 向量数据库连接失败 |
| 00004 | LLM 服务不可用 | 503 | 大语言模型服务不可用 |
| 00005 | Keycloak 服务不可用 | 503 | Keycloak 认证服务不可用 |

### 4.2 10xxx - 系统配置模块

| 异常码 | 描述 | HTTP 状态码 | 说明 |
|--------|------|-------------|------|
| 10001 | 系统已初始化 | 400 | 尝试重复初始化系统 |
| 10002 | 管理员密码长度不足 | 400 | 密码长度不符合要求 |
| 10003 | 配置文件写入失败 | 500 | 配置文件写入异常 |
| 10004 | 数据库表创建失败 | 500 | 初始化时表创建失败 |

### 4.3 11xxx - 认证模块

| 异常码 | 描述 | HTTP 状态码 | 说明 |
|--------|------|-------------|------|
| 11001 | 内置管理员已停用 | 403 | 系统已初始化后禁用内置管理员 |
| 11002 | 用户名或密码错误 | 401 | 登录凭证验证失败 |
| 11003 | Token 过期或无效 | 401 | JWT Token 验证失败 |
| 11004 | 权限不足 | 403 | 用户没有足够权限 |
| 11005 | Keycloak 认证失败 | 401 | Keycloak OAuth2 认证失败 |
| 11006 | 旧密码不正确 | 400 | 修改密码时旧密码错误 |
| 11007 | 新密码长度不足 | 400 | 新密码长度不符合要求 |

### 4.4 99xxx - 通用模块

| 异常码 | 描述 | HTTP 状态码 | 说明 |
|--------|------|-------------|------|
| 99001 | 参数校验失败 | 400 | 请求参数验证失败 |
| 99002 | 资源不存在 | 404 | 请求的资源未找到 |
| 99003 | 请求方法不允许 | 405 | HTTP 方法不支持 |
| 99004 | 请求过期 | 403 | 请求时间戳超出容忍范围 |
| 99005 | 签名验证失败 | 403 | 请求签名验证失败 |
| 99006 | 缺少签名头 | 403 | 请求缺少必要的签名头 |

## 5. 异常响应格式

### 5.1 成功响应

```json
{
  "data": {...}
}
```

### 5.2 异常响应

```json
{
  "error_code": "11001",
  "message": "Built-in admin is disabled",
  "detail": "Please use the admin account you created"
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| error_code | string | 五位数异常码 |
| message | string | 简短错误描述（英文） |
| detail | string | 详细错误信息（可选） |

## 6. 前端处理流程

```
后端返回 error_code → 前端根据 error_code 查找国际化消息 → 显示对应提示
```

### 6.1 异常码映射表

前端通过 `src/utils/errorCodes.js` 维护异常码到国际化 key 的映射：

```javascript
export const errorCodes = {
  '11001': 'errors.builtinAdminDisabled',
  '11002': 'errors.invalidCredentials',
  // ...
}
```

### 6.2 国际化消息

国际化消息分别存储在：
- `src/i18n/zh-CN/errors.js` - 中文消息
- `src/i18n/en-US/errors.js` - 英文消息

## 7. 新增异常码流程

1. 在 `backend/app/core/error_codes.py` 中添加新的 ErrorCode 枚举值
2. 在 `frontend/src/utils/errorCodes.js` 中添加异常码映射
3. 在 `frontend/src/i18n/zh-CN/errors.js` 中添加中文消息
4. 在 `frontend/src/i18n/en-US/errors.js` 中添加英文消息
5. 更新本文档

## 8. 代码安全性

- 异常码不泄露敏感信息
- 详细错误信息仅在 DEBUG 模式下返回
- 日志记录异常码便于审计
