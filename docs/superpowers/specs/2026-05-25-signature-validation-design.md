# 接口签名校验与请求时间差校验设计文档

## 1. 需求分析

### 1.1 需求描述
为所有 API 接口增加签名校验和请求时间差校验，防止接口被非法调用和重放攻击。

### 1.2 功能要求
- 签名校验：使用 HMAC-SHA256 算法对请求参数进行签名验证
- 时间差校验：验证请求时间戳与服务器时间的差值，默认允许 1 分钟
- 时间差可配置：通过环境变量配置时间差容忍度
- 可选择性排除某些接口（如登录、健康检查）

### 1.3 安全目标
- 防止接口被篡改：通过签名验证确保请求参数完整性
- 防止重放攻击：通过时间戳和随机字符串（nonce）防止请求被重复发送

## 3. 详细设计

### 3.1 配置项设计

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `SIGNATURE_ENABLED` | bool | `false` | 是否启用签名验证 |
| `SIGNATURE_SECRET_KEY` | str | - | 签名密钥（必需配置） |
| `SIGNATURE_TIMESTAMP_TOLERANCE` | int | `60` | 时间差容忍度（秒） |
| `SIGNATURE_EXCLUDED_PATHS` | str | `/api/auth/token,/health` | 排除的路径（逗号分隔） |

### 3.2 请求头设计

| 请求头 | 说明 |
|--------|------|
| `X-Sign-Timestamp` | 请求时间戳（Unix时间戳，秒） |
| `X-Sign-Nonce` | 随机字符串（防止重放攻击） |
| `X-Sign-Signature` | HMAC-SHA256 签名值（十六进制） |

### 3.3 签名算法设计

**签名生成步骤**：

1. **收集参数**：合并 URL 参数和请求体参数
2. **添加时间戳和随机串**：加入 `timestamp` 和 `nonce`
3. **字典序排序**：按键名进行字典序排序
4. **拼接字符串**：格式为 `key1=value1&key2=value2...`
5. **HMAC-SHA256 签名**：使用配置的密钥进行签名
6. **转换格式**：签名结果转为十六进制字符串

**伪代码**：

```python
def generate_signature(params: dict, secret: str) -> str:
    # 字典序排序
    sorted_keys = sorted(params.keys())
    # 拼接字符串
    sign_string = '&'.join(f"{k}={params[k]}" for k in sorted_keys)
    # HMAC-SHA256 签名
    signature = hmac.new(secret.encode(), sign_string.encode(), hashlib.sha256).hexdigest()
    return signature
```

### 3.4 验证逻辑设计

**验证步骤**：

1. **检查是否启用签名验证**：如果 `SIGNATURE_ENABLED=false`，直接跳过
2. **检查是否在排除列表**：如果请求路径在排除列表中，直接跳过
3. **检查时间戳**：验证 `timestamp` 是否在允许范围内
4. **检查签名**：重新计算签名并与请求中的签名对比
5. **返回结果**：验证通过继续处理请求，否则返回 403 错误

### 3.5 错误码设计

| HTTP 状态码 | 错误信息 | 说明 |
|------------|----------|------|
| 403 | `Signature verification failed` | 签名验证失败 |
| 403 | `Request expired` | 请求时间戳过期 |
| 403 | `Missing signature headers` | 缺少签名头 |

## 6. 安全性设计

### 6.1 密钥管理

- 签名密钥必须通过环境变量配置，禁止硬编码
- 生产环境使用强密钥（建议 32 位以上随机字符串）
- 密钥应定期轮换

### 6.2 重放攻击防护

- 通过 `nonce` 参数防止完全相同的请求被重复发送
- 通过 `timestamp` 参数限制请求的有效期

### 6.3 敏感信息保护

- 签名验证失败时不泄露具体原因（避免给攻击者提供信息）
- 记录签名验证失败日志用于审计

## 7. 部署与集成

### 7.1 后端集成

1. 在 `config.py` 中添加配置项
2. 创建签名工具类 `sign_utils.py`
3. 创建签名验证中间件 `SignatureMiddleware`
4. 在 `main.py` 中注册中间件

### 7.2 前端集成

1. 创建签名工具 `src/utils/sign.js`
2. 修改请求拦截器添加签名逻辑
3. 安装 `crypto-js` 依赖

## 8. 测试用例

### 8.1 签名验证测试

| 测试场景 | 测试步骤 | 预期结果 |
|----------|----------|----------|
| 签名正确 | 发送带有正确签名的请求 | 请求成功 |
| 签名错误 | 发送带有错误签名的请求 | 返回 403 错误 |
| 缺少签名头 | 发送不带签名头的请求 | 返回 403 错误 |
| 时间戳过期 | 发送时间戳超过容忍度的请求 | 返回 403 错误 |
| 排除路径 | 访问排除列表中的路径 | 请求成功（不验证签名） |

### 8.2 配置测试

| 测试场景 | 测试步骤 | 预期结果 |
|----------|----------|----------|
| 签名验证关闭 | 设置 `SIGNATURE_ENABLED=false` | 所有请求不验证签名 |
| 时间差配置 | 修改 `SIGNATURE_TIMESTAMP_TOLERANCE` | 按新配置验证时间戳 |

## 9. 代码安全性

### 9.1 注意事项

1. **密钥安全性**：确保密钥不被泄露
2. **参数处理**：正确处理特殊字符和编码问题
3. **错误处理**：避免泄露敏感信息
4. **日志审计**：记录签名验证失败事件

### 9.2 最佳实践

- 使用环境变量管理敏感配置
- 签名验证失败时记录日志但不返回详细错误信息
- 定期轮换签名密钥
