# 企业智能问答助手 - 使用说明

## 简介

企业智能问答助手是一个基于 Claude Code Skill 的智能问答系统，能够同时查询 SQLite 数据库和 Markdown 知识库，回答企业相关问题。

## 使用方法

### 基本调用

```bash
/enterprise-qa "您的问题"
```

### 支持的问题类型

1. **数据库查询**：员工信息、项目记录、考勤、绩效等
2. **知识库查询**：公司制度、规范、FAQ 等
3. **混合查询**：需要同时查数据库和知识库的问题
4. **边界情况**：不存在的数据、SQL注入尝试等

### 示例 2：计算年假天数

**问题**：张三理论有多少年假

**回答**：
张三的理论年假为 **6 天**。

**计算依据：**

| 项目 | 数据 |
|------|------|
| 入职日期 | 2023-06-15 |
| 当前日期 | 2026-03-27 |
| 工龄 | 2 年 |

**年假规则：**
- 入职满 1 年享 5 天
- 每增 1 年 +1 天
- 上限 15 天

**计算过程：**
- 满 1 年：5 天
- 额外 1 年（2 年 - 1 年）：+1 天
- **总计：6 天**

> 来源：employees 表 (employee_id: EMP-001, hire_date) + hr_policies.md §请假类型

---

### 示例 3：晋升条件分析

**问题**：张三目前职级是什么？可以晋级吗？

**回答**：
张三目前的职级是 **P6（高级工程师）**。

**晋升分析（P6 → P7）：**

| 条件 | 要求 | 张三情况 | 结果 |
|------|------|---------|------|
| 工作年限 | P6 满 2 年 | 入职 2 年 9 个月 | ✓ |
| 绩效要求 | 连续 4 季度 KPI≥90 或年度 2 个 S | Q1:88, Q2:92, Q3:87, Q4:90（仅 2 季度≥90） | ✗ |
| 项目经验 | 主导项目≥2 个 | 主导 2 个（ReMe 记忆框架、数据分析平台） | ✓ |
| 技术贡献 | 技术突破/专利/论文至少 1 项 | 数据库无记录 | ? |

**结论：张三目前不符合 P6→P7 晋升条件。**

主要差距：
1. **绩效不达标**：需要连续 4 季度 KPI≥90，但张三只有 Q2（92）和 Q4（90）达标
2. **技术贡献**：需要技术突破、专利或论文，数据库中无相关记录

> 来源：employees 表 + performance_reviews 表 + project_members 表 + promotion_rules.md §P6→P7

## 配置说明

### 环境变量

```bash
export ENTERPRISE_QA_DB_PATH="./enterprise.db"
export ENTERPRISE_QA_KB_PATH="./knowledge"
```

### 数据库初始化

```bash
cd interview-exam/enterprise-qa-data
chmod +x init_db.sh
./init_db.sh
```

## 技术特性

- 意图识别：自动判断查询类型
- 参数化 SQL：防止 SQL 注入
- 知识库索引：BM25 或 embedding 检索
- 多源融合：自动整合数据库和知识库结果
- 来源标注：每个回答都标注数据来源

## 评分标准

- **功能分（60分）**：纯数据库查询、纯知识库查询、混合查询、边界情况
- **技术分（25分）**：代码质量、安全性、单元测试覆盖率
- **设计分（15分）**：可扩展性、引用标注、创新性
